## Telegram Lead Bot

Telegram бот для сбора заявок из магазина.  
Бот задаёт 3 вопроса, показывает подтверждение и отправляет заявку менеджеру (в личку или в групповой чат).

### Технологии

- **Node.js**
- **Telegraf**
- **dotenv**

### Структура проекта

- **bot.js** — основной файл бота
- **package.json** — настройки проекта и зависимости
- **.env** — переменные окружения
- **README.md** — документация

### Переменные окружения (.env)

Создайте файл `.env` в корне проекта со следующим содержимым:

```bash
BOT_TOKEN=YOUR_TELEGRAM_BOT_TOKEN
MANAGER_CHAT_ID=YOUR_MANAGER_CHAT_ID
```

- **BOT_TOKEN** — токен бота из BotFather
- **MANAGER_CHAT_ID** — ID менеджера или группового чата, куда будут отправляться заявки  
  (можно узнать, добавив бота в чат и посмотрев входящие апдейты или с помощью сторонних инструментов).

2. **Установите зависимости** (уже выполнено, но на всякий случай):

```bash
npm install
```

3. **Заполните `.env`** корректными значениями `BOT_TOKEN` и `MANAGER_CHAT_ID`.

### Запуск бота

Запустите бота командой:

```bash
node bot.js
```

или через npm-скрипт:

```bash
npm start
```

### Логика работы бота

1. **Команда `/start`**
   - Бот отправляет приветствие:
     - «Здравствуйте! 👋»
     - «Добро пожаловать в магазин [Название].»
     - «Чтобы оставить заявку, ответьте на 3 вопроса.»
   - Показывается инлайн-кнопка: **`✅ Начать`**

2. **Шаг 1 — Имя**
   - После нажатия на **`✅ Начать`**:
   - Бот спрашивает: **«Как вас зовут?»**
   - Ответ сохраняется в `ctx.session.name`.

3. **Шаг 2 — Телефон**
   - Бот спрашивает:
     - «Отправьте номер телефона»
     - «(пример: +998901234567)»
   - Показывается обычная клавиатура с кнопкой:
     - **`📲 Отправить мой номер`** (используется `request_contact: true`)
   - Бот принимает:
     - контакт (`ctx.message.contact`)
     - текстовый номер, введённый вручную
   - Значение сохраняется в `ctx.session.phone`.

4. **Шаг 3 — Что интересует**
   - Бот спрашивает:
     - «Что вас интересует?»
     - Примеры:
       - ванна
       - плитка
       - смеситель
       - унитаз
   - Ответ сохраняется в `ctx.session.interest`.

5. **Шаг 4 — Подтверждение**
   - Бот отправляет сводку:

     ```text
     Проверьте заявку:

     Имя: {name}
     Телефон: {phone}
     Интересует: {interest}
     ```

   - Показываются кнопки:
     - **`✅ Отправить заявку`**
     - **`✏️ Изменить`** (начать заново с шага «Имя»)

6. **Шаг 5 — Отправка менеджеру**
   - В чат с ID `MANAGER_CHAT_ID` отправляется сообщение:

     ```text
     📩 Новая заявка

     Имя: {name}
     Телефон: {phone}
     Интересует: {interest}

     Telegram: @{username} / или «нет username»
     ID: {user_id}
     ```

7. **Шаг 6 — Ответ клиенту**
   - После успешной отправки менеджеру бот пишет пользователю:
     - «Спасибо! ✅»
     - «Заявка отправлена.»
     - «Менеджер свяжется с вами в ближайшее время.»

### Хранение данных

- Используется **сессионное хранилище** Telegraf:

```js
ctx.session = {
  name: '',
  phone: '',
  interest: '',
  step: null,
};
```

- Данные хранятся временно в памяти и позволяют работать с несколькими пользователями одновременно.

### Обработка ошибок и неожиданных сообщений

- Бот:
  - корректно обрабатывает случаи, когда пользователь отправляет неожиданные сообщения;
  - просит использовать кнопки на этапе подтверждения;
  - предлагает начать сначала через `/start`, если состояние нарушено.

### Дополнительно

- Код написан на **Node.js** с использованием **Telegraf** и **dotenv**.
- Все тексты бота — на **русском языке**.
- Бота можно разворачивать на любом хостинге/сервере, где доступен Node.js и есть доступ к Telegram API.
